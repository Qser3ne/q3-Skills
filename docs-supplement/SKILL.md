# Document Supplement

用于修改、补充、整理已有文档时，避免把新增内容直接追加到文档末尾。目标是让文档像一次完整修订后的成稿，而不是多次补丁堆叠。

## Workflow

1. 先阅读现有文档，识别它的主题、标题层级、已有知识点、重复内容和缺口。
2. 判断新增内容应归入哪个已有章节或知识点。
3. 基于新增内容重写相关段落：
   - 修正旧内容中过时、不准确或表述不完整的部分。
   - 删除与新增内容重复或语义重叠的内容。
   - 补全相关知识点，使上下文连贯。
   - 调整标题、段落顺序和过渡句，让结构自然。
4. 只有当新增内容与既有主题、标题和知识点完全无关时，才新增标题或章节。
5. 完成后检查整篇文档是否仍然像统一作者一次性整理出的内容。

## Editing Rules

- 不要默认在文档末尾新增“补充”“新增内容”“附加说明”等章节。
- 不要保留同一知识点的多份近似解释。
- 不要只局部插入一句话而忽略前后段落是否需要重写。
- 如果新增内容改变了旧结论，必须同步改写旧结论。
- 如果新增内容只是细化已有知识点，应并入原章节，而不是新建同级标题。
- 如果需要新增标题，标题应反映真实主题，不应命名为“补充内容”。

## Output Expectations

- 输出应是修订后的完整相关文档内容或清晰的补丁方案，取决于用户要求。
- 当无法确定新增内容应合并到哪里时，先说明候选位置和推荐位置，再继续处理。
- 保持原文已有风格、语言、标题层级和格式约定，除非它们阻碍文档清晰度。
