# 主树模板

把它作为 `ctf-worklog/tree.md` 的默认形态。文件名保留 `tree.md` 是为了兼容旧工作流，内容实际维护任务-结论关系图。

## 规则

- 主文档分成 `## 任务` 与 `## 结论` 两部分。
- 任务序号和结论序号分别独立递增，且都从 `1` 开始。
- `任务 1` 固定为环境基线采集任务，`结论 1` 固定为环境基线结论。
- 只有 `已确认` 结论可以作为后续任务的前置结论。
- 每个任务固定写成 `目标`、`前置结论`、`产出结论`、`证据`、`附加` 五行。
- 每个结论固定写成 `内容`、`产出任务`、`证据`、`后续任务` 四行。
- 真正的依赖关系只看 `前置结论`、`产出结论`、`产出任务` 和 `后续任务` 字段，不看序号大小或相邻位置。
- 任务的附录文件名绑定任务序号，例如 `./appendices/task-3.md`。

## 每轮同步检查清单

每次观察、实验、题目相关失败尝试、状态变化、结论变化或任务拆分后，先完成以下检查，再继续下一步：

- 如果阻塞原因只是 Docker Hub、靶机或其他外部基础设施连通性异常，先在对话里告知用户；不要创建任务、结论或附录，也不要进入下面的文档同步流程。

- 主文档已同步：相关任务的状态、证据、产出结论和附加链接已更新。
- 结论块已同步：新增、确认、否定或复核的结论已更新 `产出任务`、`证据` 和 `后续任务`。
- 附录已同步：需要可复现上下文、题目相关失败细节、命令、请求、payload、样本或剩余未知点时，已更新对应 `appendices/task-N.md`；外部连通性异常不落档。
- 双向引用已自检：任务的 `产出结论` 对应结论的 `产出任务`，结论的 `后续任务` 对应任务的 `前置结论`。
- 未更新文档不得继续枚举、实验、利用、切换新方向或输出阶段性结论。

## 骨架

```md
# CTF 任务-结论验证链

## 任务

任务 1 [已完成] 目标：完成目标环境的基线采集，确认入口、附件、交互方式、技术栈线索与可见约束
前置结论：无；环境基线采集任务
产出结论：结论 1
证据：入口地址、附件、题面约束、交互方式、源码/容器/文件线索等客观环境事实均已记录
附加：[appendix](appendices/task-1.md)

任务 2 [已完成] 目标：从题目附件 `payload.txt` 中提取可疑编码片段，并验证其是否能稳定解码为结构化数据
前置结论：结论 1 [已确认]
产出结论：结论 2
证据：同一片段重复解码后均得到一致的 gzip 文件头与可解压数据
附加：[appendix](appendices/task-2.md)

任务 3 [待执行] 目标：解压任务 2 得到的数据，并识别其中的文件结构、字段含义或下一阶段输入格式
前置结论：结论 2 [已确认]
产出结论：无
证据：待执行
附加：[appendix](appendices/task-3.md)

任务 4 [已完成] 目标：核验公开入口中的 `debug` 参数是否会切换到不同的响应分支
前置结论：结论 1 [已确认]
产出结论：结论 3
证据：带参和不带参请求的响应正文、状态码与耗时均无稳定差异，源码中也未发现对应分支
附加：[appendix](appendices/task-4.md)

## 结论

结论 1 [已确认] 内容：目标环境已完成基线采集，确认了入口、附件、交互方式与当前可见技术栈线索
产出任务：任务 1 [已完成]
证据：环境基线任务已记录入口、关键交互结果、可用附件或源码线索与题面约束
后续任务：任务 2，任务 4

结论 2 [已确认] 内容：附件中的编码片段可稳定解码为 gzip 压缩数据
产出任务：任务 2 [已完成]
证据：重复解码得到一致的 gzip 文件头，且解压过程未出现随机波动
后续任务：任务 3

结论 3 [已否定] 内容：公开入口中的 `debug` 参数会切换到不同的有效响应分支
产出任务：任务 4 [已完成]
证据：带参和不带参请求的结果一致，且源码核验未发现对应处理逻辑
后续任务：无
```

## 编写说明

- 宽泛方向例如“这里可能有 SQL 注入”或“这个样本可能藏着下一阶段数据”不要直接写进主树；先改写成足够小、可独立执行的任务。
- 已得到的事实写成结论，下一步验证工作写成任务。
- 一个任务产出多个彼此独立的事实时，拆成多个结论。
- 多个任务支撑同一事实时，更新已有结论的 `产出任务` 与 `证据`，不要重复造相同结论。
- 如果任务还没做完，但已有中间材料，把长推理、原始请求、终端输出、样本片段或题目相关失败尝试写进附录，不要提前把任务改成 `已完成`。
- 如果失败只是 Docker Hub、靶机或其他外部基础设施连通性异常，直接告知用户，不要写进主树或附录。
